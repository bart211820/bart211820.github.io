Hoe ga ik aan de slag?

Papier

Meestal voordat ik ga beginnen met het programmeren van een website of applicatie, teken ik eerst snel op papier een eerste ontwerp voor hoe ik het er uit wil laten zien. Dit geeft mij een goed beeld voor wat ik later moet uit gaan programmeren en maakt het programmeer proces sneller.

Ook als je met meerdere mensen aan een project werkt of iets voor een klant maakt is het zeker handig om eerst een ontwerp te hebben en dit dan met de andere betrokkenen te bespreken. Zodat er geen schermen worden geprogrammeerd worden waar iemand later niet blij mee is.

Hieronder is foto te zien van het papier ontwerp dat ik voor deze website heb gemaakt voordat ik de html ging schrijven.

Wireframes

Afhangend van hoe het paieren ontwerp is gegaan maak ik nog een ontwerp in de vorm van een wireframe. Dit is netter en meer gedetailleerd en geeft hierdoor een duidelijk beeld op de uiteindelijke pagina. Ook kan je met de hulp van sommige programma's al links plaatsen waardoor je al van pagina naar pagina kan navigeren ondanks dat je eigenlijk nog niks hebt hoeven programmeren.

Aangezien ik al een goed beeld had van deze website na mijn papier ontwerp heb ik er geen wireframe voor gemaakt. Echter heb ik hier wel een ander voorbeeld van een auto verhuur website voor een school project van eerder dit jaar.

Code

Zodra de ontwerpen af zijn en alles is overlegd met de eventuele betrokkenen, begin ik met het programmeren. Voor websites gebruik talen zoals html, css en javascript en voor applicaties gebruik ik persoonlijk het liefst java ookal kan ik ook met andere talen programmeren. Voor een volledige lijst bekijk mijn vaardigheden pagina. En uiteindelijk zal er een product zoals bijvoorbeeld deze website als eindproduct komen.
